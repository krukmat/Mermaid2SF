# Formatting & Schemas

## Preferred formats by role
- Functional / PO: Markdown with explicit sections + ACs in Given/When/Then.
- Architecture: Markdown + tables for tradeoffs; add ADR skeletons where decisions matter.
- QA: Tables for test cases and risk matrix; separate manual vs automation candidates.

## Minimal sections (always include)
- Scope
- Out of Scope
- Assumptions
- Risks
- Open Questions
- Next Actions

## Determinism
- When producing lists/tables, keep ordering stable and explain prioritization criteria.
